## 📝 使用说明

### 添加新待办事项

当发现新问题或想改进时，按以下格式添加：

```markdown
- [ ] 简短描述问题
  - 详细说明
  - 发现日期：YYYY-MM-DD
  - 优先级：🔴高 / 🟡中 / 🟢低
```

### 完成待办事项

完成后，移到 `CHANGELOG.md` 并在前面打勾：

```markdown
- [x] 已完成的任务
```

---

## 🔍 快速检查清单

每次修改网站前，确认：

- [ ] 备份当前工作（git status 查看是否有未提交的改动）
- [ ] 在本地测试修改效果
- [ ] 确认不影响现有功能
- [ ] 提交到 git 并写清楚 commit message
- [ ] 推送到 GitHub 后确认在线效果
